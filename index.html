🎯项目概述

这是一个革命性的UI自动化测试框架，它将传统的Appium自动化测试与现代AI技术完美结合， 创造出一个能够智能选择驱动、自动生成代码、支持自然语言操作的下一代测试解决方案。

🌟 项目愿景

让每个测试工程师都能享受AI带来的便利，用自然语言描述测试步骤，让AI自动选择最佳的执行方式， 并自动生成可维护的测试代码。从此告别复杂的元素定位和繁琐的代码编写。

🎯 核心价值

🧠 智能化

AI自动分析操作复杂度，智能选择Appium或MidScene驱动，无需人工干预

🗣️ 自然语言

用自然语言描述测试步骤，如"点击登录按钮"、"输入用户名"，AI自动理解并执行

📝 自动生成

AI执行后自动生成标准的测试用例和页面对象代码，保存到Android目录

🔄 无缝切换

在同一个测试流程中，AI可以无缝切换不同驱动，确保每个操作都用最佳方式执行

🏆 技术优势

  • 零学习成本：测试人员无需学习复杂的API，用自然语言即可编写测试
  • 高成功率：智能驱动选择 + 备用方案，确保操作执行成功
  • 自动维护：AI自动生成和维护页面对象，减少维护工作量
  • 标准化输出：生成的代码符合行业标准，便于团队协作

✨核心特性

🤖 AI智能驱动选择

SmartDriverSelector 是框架的大脑，它能够：
  • 分析操作描述的复杂度（简单点击 vs 复杂逻辑）
  • 计算AI处理的信心度（高信心度用MidScene，低信心度用Appium）
  • 实时性能监控和优化建议
  • 自动学习和调整选择策略
# 智能选择示例
"点击登录按钮" → AI信心度: 95% → 选择MidScene
"循环检查页面元素直到出现" → 复杂度: 高 → 选择Appium
"如果弹窗出现则点击确定" → 条件逻辑 → 选择Appium

🔄 无缝驱动切换

这是框架最独特的功能之一。在同一个测试流程中，AI可以根据每个操作的特点自动切换驱动：

# 实际测试流程示例
mc.run_ai_action("启动驾考宝典应用")        # → Appium (精确启动)
mc.run_ai_action("点击登录按钮")            # → MidScene (AI视觉)
mc.run_ai_action("循环检查页面元素")        # → Appium (复杂逻辑)
mc.run_ai_action("输入手机号13800138000") # → MidScene (简单输入)

🐍 Python编程接口

from mc_ui_auto import mc

# AI自动执行操作
mc.run_ai_action("启动驾考宝典应用")
mc.run_ai_action("点击登录按钮")
mc.run_ai_action("输入用户名test@example.com")

# AI查询和断言
result = mc.run_ai_query("获取当前页面标题")
mc.run_ai_assert("页面显示登录成功")

# 生成测试代码
mc.generate_test_code("login_test")
