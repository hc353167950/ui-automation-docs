💡 使用指南

完整的使用方法和最佳实践

🎯三种使用方式

🎮 方式1: MCP工具使用（推荐）

最佳体验方式：通过AI Assistant直接调用MCP工具 
# 1. 启动MCP服务器
python start_mcp_server.py

# 2. 在AI Assistant中使用
/ai_action action_text="启动驾考宝典应用"
/ai_action action_text="点击登录按钮"
/ai_action action_text="输入手机号13800138000"
/generate_test_code task_name="登录测试"

⚡ 方式2: 命令行运行

# 智能驱动模式（推荐）
python run.py --driver smart

# 指定驱动类型
python run.py --driver appium     # 传统Appium
python run.py --driver midscene   # MidScene AI

# 运行指定测试
python run.py -f Android/test_case/test_login.py

# 详细输出
python run.py -v

🐍 方式3: 编程接口

from mc_ui_auto import mc

# AI自动执行操作
mc.run_ai_action("启动驾考宝典应用")
mc.run_ai_action("点击登录按钮")
mc.run_ai_action("输入用户名test@example.com")

# AI查询和断言
result = mc.run_ai_query("获取当前页面标题")
mc.run_ai_assert("页面显示登录成功")

# 生成测试代码
mc.generate_test_code("login_test")

🏆最佳实践

📝 操作描述建议

✅ 推荐写法

# 清晰具体的描述
"点击登录按钮"
"在用户名输入框输入admin"
"滑动到页面底部"
"点击红色的确认按钮"

# 包含条件的复杂操作
"如果有权限弹窗，点击允许"
"等待页面加载完成，然后点击下一步"

❌ 避免写法

# 模糊不清的描述
"点击那个按钮"
"输入一些文字"
"做一些操作"

# 过于复杂的逻辑
"循环检查列表中的每个元素，如果找到包含特定文本的项目，则点击它，否则滚动到下一页继续查找"

🎯 任务组织建议

建议将复杂的测试流程分解为多个简单的步骤，每个步骤专注于一个具体的操作， 这样AI可以更好地选择合适的驱动，提高执行成功率。
