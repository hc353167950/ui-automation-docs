🎛️MCP服务器架构

Model Context Protocol (MCP) 是连接AI和自动化框架的桥梁， 让AI能够直接调用框架的37个专业工具，实现真正的AI驱动测试。

🏗️ 架构特点

  • 标准协议：基于JSON-RPC的标准MCP协议
  • 高性能：使用FastMCP实现，支持并发调用
  • 智能路由：自动选择最佳驱动执行操作
  • 错误处理：完善的错误处理和重试机制
  • 日志记录：详细的操作日志和性能监控

🛠️MCP工具详解

🤖 AI操作工具

  • /ai_action
    执行AI自动化操作，智能选择驱动
  • /ai_query
    执行AI查询，获取页面信息
  • /ai_assert
    执行AI断言验证
  • /ai_wait_for
    执行AI等待操作

📝 代码生成工具

  • /generate_test_code
    生成测试代码到Android目录
  • /get_code_generation_summary
    获取代码生成摘要
  • /clear_operation_records
    清空操作记录

📱 设备操作工具

  • /get_page_elements
    获取页面元素信息
  • /screenshot
    获取设备截图
  • /get_device_info
    获取设备信息
  • /get_current_activity
    获取当前Activity

🔧 系统工具

  • /health_check
    健康检查
  • /get_capabilities
    获取驱动能力
  • /restart_driver
    重启驱动
  • /get_driver_status
    获取驱动状态

💡完整使用示例

🎮 登录测试完整流程

# 1. 启动MCP服务器
python start_mcp_server.py

# 2. 在AI Assistant中执行测试流程
/ai_action action_text="启动驾考宝典应用"
/ai_action action_text="点击登录按钮"
/ai_action action_text="输入手机号13800138000"
/ai_action action_text="点击获取验证码"
/ai_wait_for condition_text="等待验证码输入框出现"
/ai_action action_text="输入验证码1234"
/ai_action action_text="点击登录"
/ai_assert assertion_text="页面显示登录成功"

# 3. 生成测试代码
/generate_test_code task_name="手机号登录测试" task_steps=[
    "启动应用",
    "点击登录",
    "输入手机号",
    "获取验证码",
    "输入验证码",
    "完成登录",
    "验证成功"
]
